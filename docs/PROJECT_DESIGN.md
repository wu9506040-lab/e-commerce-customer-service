## 7. 模型能力边界（关键讨论点）

### LLM 能做（强项）

| 能力 | 适用 | 备注 |
|---|---|---|
| 自然语言理解 | ✅ 用户问题分类 | 准确率 ~95% |
| 文本生成 | ✅ 客服回答 | 流畅自然 |
| 多轮对话 | ✅ 上下文理解 | 受 context window 限制 |
| 文档总结 | ✅ 长文档压缩 | 适合 FAQ 整理 |

### LLM 不能做（弱项 / 禁止依赖）

| 能力 | 现状 | 替代方案 |
|---|---|---|
| 精确计算 | ❌ 价格 / 折扣计算 | 必须程序逻辑 |
| 实时数据 | ❌ 库存 / 物流状态 | 必须查 MySQL / 外部 API |
| 严格格式 | ❌ JSON / 表单生成 | 用 function calling |
| 多步推理 | ⚠️ 受 prompt 影响 | 单步任务可靠，链式易错 |
| 事实记忆 | ❌ 跨会话记忆 | 用 Redis / MySQL 持久化 |
| 工具调用 | ⚠️ Agent 模式（V2.0）| 当前禁用 |

### 失败模式（容错策略）

| 场景 | LLM 表现 | 系统应对 |
|---|---|---|
| 召回无相关知识 | 幻觉生成 | prompt 约束 + 强制"我不知道" |
| LLM 超时 | 504 | 重试 1 次 + 降级到 FAQ |
| embedding API 挂 | 检索失败 | 缓存 + 离线 fallback |
| LLM token 超限 | 中断 | 截断历史 + 滑动窗口 |

### 模型选择策略
