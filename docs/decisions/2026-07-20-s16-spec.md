## 1. 退款服务（RefundService）

### 1.1 接口契约（`app/services/refund/protocols.py`）

```python
@runtime_checkable
class RefundService(Protocol):
    """退款服务协议（CLAUDE.md §9.3.2 支持模块替换）

    V2 范围：仅做读（list / get / get_status），写操作（create / update）留 V3+
    —— YAGNI：当前业务场景不需要 AI 自动创建/审批退款（需人工介入审核）。
    接入方：自建商城/OA/ERP 实现此接口即可对接 AI 客服的退款查询。
    """

    async def get_refund(self, user_id: int, refund_no: str) -> Optional[Refund]:
        """按退款号查退款详情（含 order_no 关联，强制 user_id 防越权）；不存在返 None"""
        ...

    async def list_user_refunds(
        self, user_id: int, status: Optional[str] = None,
        start_date: Optional[datetime] = None, end_date: Optional[datetime] = None,
        limit: int = 20, cursor: Optional[str] = None,
    ) -> tuple[List[Refund], Optional[str]]:
        """查用户退款列表；返 (refunds, next_cursor)；cursor 为下一页游标（无更多则 None）"""
        ...

    async def get_refund_status(self, refund_no: str) -> Optional[str]:
        """查退款当前状态（pending / approved / rejected / completed）；不存在返 None"""
        ...
```

## 2. 物流服务（LogisticsService）

### 2.1 接口契约（`app/services/logistics/protocols.py`）

```python
@runtime_checkable
class LogisticsService(Protocol):
    """物流服务协议（CLAUDE.md §9.3.2 支持模块替换）

    V2 范围：仅查询（query / track），不支持下单/取消（V3+ 留）。
    当前默认实现：Mock（基于订单状态生成 mock 轨迹，无真实物流表）。
    接入方：自建商城/ERP/快递100 实现此接口即可对接 AI 客服的物流查询。
    """

    async def query(self, order_no: str) -> Optional[Logistics]:
        """按订单号查物流汇总（carrier + 状态 + 最新位置）；订单不存在返 None"""
        ...

    async def track(self, tracking_no: str) -> Optional[TrackingInfo]:
        """按运单号查完整轨迹（events 列表）；运单号无效返 None"""
        ...

    async def get_carriers(self) -> List[str]:
        """支持的快递公司列表（V2 mock 返 ['顺丰', '中通', '圆通', '韵达']）"""
        ...
```

### 2.2 Schema（`app/schemas/business.py` 追加）

```python
class TrackingEvent(BaseModel):
    """物流轨迹单事件"""
    time: datetime
    event: str                           # 已下单 / 已发货 / 运输中 / 已签收 / 已退回
    location: Optional[str] = None


class Logistics(BaseModel):
    """物流汇总（query 返回）"""
    order_no: str
    tracking_no: Optional[str] = None
    carrier: Optional[str] = None        # 顺丰 / 中通 / etc
    status: str                          # 待发货 / 运输中 / 已签收 / 已退回 / 未知
    last_location: Optional[str] = None
    estimated_arrival: Optional[datetime] = None


class TrackingInfo(BaseModel):
    """物流轨迹详情（track 返回）"""
    tracking_no: str
    carrier: str
    status: str
    events: List[TrackingEvent] = Field(default_factory=list)
```

### 2.3 Mock 默认实现（`app/services/logistics/mock_impl.py`）

**为什么是 Mock 而非 MySQL**：当前 ORM 没有 logistics 表（V2 不持久化物流轨迹，每次根据订单状态实时生成）。

```python
class MockLogisticsService:
    """Mock 实现：基于 OrderService.query(order_no) 拿订单状态 → 生成 mock 物流信息"""

    def __init__(self, order_service: Optional[OrderService] = None):
        self._order_service = order_service or get_order_service_factory().get_order_service()

    async def query(self, order_no):
        # 调 OrderService.get_order(不传 user_id，需用 get_order_status 兜底)
        #   ↓
        # 实际上 OrderService.get_order 强制要 user_id；这里要新方法 get_order_for_logistics
        #   ↓
        # **决策**：复用 OrderService.get_order，但需要传 user_id
        #   ↓
        # **替代方案**：新增 get_order_by_no_no_user_check（V2 不做；Mock 直接查 ORM）
        # **V2 最终方案**：MockLogisticsService 内部直接查 Order ORM（不走 OrderService）
        #   原因：物流查询不涉及 user 越权（订单号本身就是公开 token）
        #   符合 §9.2.2 — Mock 是基础设施层，可以查 ORM

    async def track(self, tracking_no):
        # 从 tracking_no 反推 order_no（格式：SF{order_no[3:]})
        # → query(order_no) → 包成 TrackingInfo

    async def get_carriers(self):
        return ['顺丰', '中通', '圆通', '韵达']
```

**关键决策**（spec §2.3）：
- Mock 不强制走 OrderService（避免循环依赖）
- Mock 直接查 Order ORM（基础设施层查 ORM 是允许的，§9.2.2）
- 接入方真实实现（HTTP 调快递100 API）时，自然不涉及这个问题

### 2.5 重构 Tool（`app/tools/order_tool.py`）

**只重构 `get_logistics` 方法**：

```python
# 重构前：
@staticmethod
def get_logistics(order_no: str) -> dict:
    # 直接查 ORM + 硬编码状态映射 + 手搓 trajectory
    ...

# 重构后：
@staticmethod
def get_logistics(order_no: str) -> dict:
    svc = get_logistics_service_factory().get_logistics_service()
    logistics = run_sync(svc.query(order_no))
    return OrderTool._logistics_dto_to_dict(logistics) if logistics else {
        "order_no": order_no, "logistics_no": None,
        "status": "订单不存在", "last_location": None, "trajectory": [],
    }
```

**`get_order_items(order_id)` 不重构**（Sprint 15 已说明：order_items 表无独立 Protocol 抽象空间，留 V3+）。

**公开签名 100% 不变**：`get_logistics(order_no) -> dict` 入参和返回 dict 字段不变。

## 5. 8 件套交付（CLAUDE.md §9.8）

1. 模块职责：learning_log §56（退款）+ §57（物流）
